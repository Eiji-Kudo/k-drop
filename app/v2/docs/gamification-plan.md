# v2 ゲーミフィケーション計画

- 作成日: 2026-04-05
- ステータス: Draft

## 目的

`app/v2` を「見た目がかわいいクイズアプリ」で終わらせず、遊ぶほど成長実感と再挑戦意欲が高まる体験にする。

この計画は、UI/UX ブラッシュアップ計画を補完するものとして、主に以下を設計対象にする。

- 日次で戻ってきたくなる理由
- 1 セッション終わったあとにもう 1 回やりたくなる理由
- 自分の成長を貯めていく気持ちよさ
- 他ユーザーとの比較で熱量が上がる見せ方

## 現時点の前提

### 既存ドメインで使えるもの

現行の `v2` スキーマには、すでにモチベーション設計に使える土台がある。

- `score_tiers`
  - 総合 / グループ別の tier を持てる
- `user_score_states`
  - 現在スコア、正答数、回答数、最終回答日時を保持できる
- `user_score_snapshots`
  - 日次の成長推移を見せられる
- `leaderboard_snapshots` / `leaderboard_entries`
  - 全体 / グループ別のランキングを持てる
- `quiz_sessions` / `quiz_answers`
  - セッション進行、正答数、獲得スコアを扱える
- `drop_wallets` / `drop_transactions`
  - 報酬通貨を将来的に絡められる

### 今はまだないもの

以下は現スキーマに専用テーブルがなく、必要なら後で追加する。

- 連続プレイ streak の永続化
- デイリーチャレンジ / 週間ミッション
- 実績解除ログ
- バッジ定義の永続管理
- シーズン制やイベント限定報酬

## 外部調査からの示唆

### 1. 進捗は「早い初速」と「近いゴール」が重要

調査メモ:

- Progress 表示は有効だが、序盤が重く見えると逆に離脱を招きやすい
- 大きすぎるゴールは避け、短いマイルストーンに分割した方がよい
- 「もう始めている感」を与えると継続率が上がる

設計への反映:

- ホームでは「次レベルまであと少し」を見せる
- クイズ中はセッション全体だけでなく、1 問ごとの小さな前進を返す
- 結果画面は総括だけでなく、「あと何点」「あと 1 回」で届く近い目標を出す

### 2. streak は効くが、壊れたときに一気に冷める

調査メモ:

- streak は日次習慣の強い動機になる
- ただし broken streak を強く突きつけると、そのまま離脱の引き金になりやすい

設計への反映:

- いきなり厳格な連続日数より、「今週 3 回達成」のような緩い継続指標を優先する
- streak を導入する場合も、失敗演出を強くしすぎない
- 将来的に入れるなら repair や grace の概念を検討する

### 3. leaderboard は単体だと危険

調査メモ:

- leaderboard は競争、目標設定、即時フィードバックには有効
- 一方で下位ユーザーを冷やすリスクがある
- 相性差が大きく、雑に全体順位だけ見せると逆効果になりうる

設計への反映:

- 全体 Top ランキングだけでなく「自分の周辺順位」を重視する
- 絶対順位だけでなく「次に抜ける相手」「同 tier 内の進捗差」を見せる
- ランキングは成長導線とセットで扱い、敗北感だけ残さない

## 目指す体験ループ

### ループ 1: Daily Return

1. ホームを開く
2. 今日の目標や近い報酬が見える
3. 1 タップでクイズへ入る
4. 小さな成果を得る
5. ホーム / プロフィール / ランキングで反映を見る

狙い:

- 毎日開く理由を作る
- 「短時間でも進む」と感じさせる

### ループ 2: Session Momentum

1. グループを選ぶ
2. 問題を解く
3. 正誤と加点が返る
4. セッション結果で成長が見える
5. 「もう 1 回」「別グループで挑戦」に自然につながる

狙い:

- 1 セッションの途中離脱を減らす
- 1 セッション終了後の再挑戦率を上げる

### ループ 3: Long-term Identity

1. 累積スコアが伸びる
2. tier / 称号 / バッジが変わる
3. プロフィールで蓄積を眺める
4. ランキングで比較する
5. さらに伸ばしたくなる

狙い:

- 短期報酬だけでなく、積み上げの意味を作る
- K-POP ファンとしての自己表現につなげる

## MVP で先に入れるべき要素

### A. Home Motivation Layer

- [ ] 今日のおすすめ導線
- [ ] 次 tier までの進捗カード
- [ ] 直近で伸びた指標の表示
- [ ] 「あと 1 プレイで届く」短期目標

モック段階の表現例:

- `次のレベルまで 120pt`
- `今週 2 / 3 回プレイ`
- `LE SSERAFIM 力が昨日より +80`

### B. Quiz Reward Layer

- [ ] 正解時に加点感を返す
- [ ] 連続正解感を返す
- [ ] セッション進捗を見せる
- [ ] セッション完了時に獲得結果をまとめる

モック段階の表現例:

- `+120 Score`
- `2問連続正解`
- `あと 2 問で完了`
- `今回の挑戦で tier 進捗が 14% 上昇`

### C. Result Screen Loop-back

- [ ] 今回の獲得スコア
- [ ] tier 進捗更新
- [ ] 上がった指標
- [ ] 再挑戦導線
- [ ] 別グループ挑戦導線

重要ポイント:

- 結果画面は「終了」ではなく「次の挑戦に繋ぐ画面」にする

### D. Ranking Motivation Layer

- [ ] Top 3 の見せ場
- [ ] 自分の周辺順位
- [ ] 次に抜ける相手
- [ ] 同 tier 内比較

重要ポイント:

- いきなり最上位と比較させず、近い相手を基準にする

### E. Profile Growth Layer

- [ ] 現在 tier
- [ ] 次 tier までの残り
- [ ] バッジ / 称号の獲得感
- [ ] グループ別の得意領域
- [ ] 最近の伸び

重要ポイント:

- 単なるステータス一覧ではなく、「積み上がっている感」を出す

## UI に落とすときの具体策

### Home

- 主 CTA の近くに短い報酬文脈を置く
- 大目標より短期目標を優先表示する
- 進捗バーは長大な 1 本ではなく、小さい達成単位に分ける

### Quiz

- 正解 / 不正解だけで終わらせず、獲得と前進を返す
- 解説の前に感情フィードバック、解説の後に次アクションを置く
- セッション終盤ほど「あと少し」が分かるようにする

### Ranking

- `Top Rank` と `Around You` を分ける
- 競争を煽るだけでなく「届きそう感」を作る

### Profile

- 成長曲線よりも先に「今の位置」と「次の目標」を見せる
- バッジは一覧よりも獲得ストーリーを感じられるようにする

### Phase 0: モックだけで検証する

- [ ] ホームに短期目標カードを追加
- [ ] クイズ結果に gained score を追加
- [ ] プロフィールに次 tier までの残りを追加

成功条件:

- 何も永続化しなくても「続ける理由」が UI 上で感じられる

### Phase 1: 既存スキーマで成立させる

既存テーブルから導出可能なもの:

- [ ] current tier
- [ ] next tier までの残り
- [ ] 直近のスコア増加
- [ ] グループ別成長
- [ ] ランキング差分

この段階では新規テーブルを増やさず、既存スキーマの価値を最大化する。

### ホーム

- [ ] まず押すべき導線を 1 つに絞る
- [ ] 今日の目標カードを置く
- [ ] 次 tier 進捗を置く
- [ ] 直近の成果をチップ表示する

### クイズ導線

- [ ] グループ選択時に期待報酬を見せる
- [ ] 問題中に session progress を見せる
- [ ] 回答後に gained score / combo を返す

### 結果画面

- [ ] 今回の成果サマリー
- [ ] 成長した項目
- [ ] 次アクション CTA 2 種

### ランキング

- [ ] Top 3 強調
- [ ] around-you 表示
- [ ] 次に抜ける相手の差分表示

### プロフィール

- [ ] current tier / next tier
- [ ] 収集と成長の両方を見せる
- [ ] グループ別 mastery を見せる

## 失敗しやすい設計

- 全部を streak に頼る
- leaderboard を全体順位だけで見せる
- 遠すぎる目標を大きく見せる
- 報酬演出だけ派手で、意味のある成長が見えない
- 負けた / 途切れた体験を強く見せすぎる

## 検証したい指標

モック段階で最低限見るべきもの:

- ホームからクイズ開始までの迷いの少なさ
- クイズ結果画面からの再挑戦意欲
- プロフィールで自分の成長が伝わるか
- ランキングを見て嫌になるより「届きそう」と思えるか

将来的に計測したいもの:

- クイズ開始率
- セッション完了率
- 結果画面からの再挑戦率
- D1 / D7 継続率
- グループ別の再訪率
