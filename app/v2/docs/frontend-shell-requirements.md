# v2 フロントエンド ガワアプリ 要件

## 目的

モックデータで全画面を繋ぎ、UI/UX の手触りを確認する。バックエンド API 接続は後のフェーズで行う。

## 技術スタック

| 技術 | 用途 |
|---|---|
| TanStack Router | ファイルベースルーティング |
| TanStack Query | データ取得（モックデータ） |
| React Hook Form + Zod | フォームバリデーション |
| **Tailwind CSS + daisyUI** | **UI フレームワーク（valentine テーマ）** |
| Orval | **後回し**（API 接続時に導入） |

## デザイン方針

- **daisyUI valentine テーマ** をベースに、v1 のカラーパレットに合わせてカスタマイズ
- モバイルファースト（PWA 想定）
- カードベースのレイアウト
- v1 のピンク・紫系カラーパレットを踏襲（Primary: #FF9FCC, Accent: #A855F7, Background: #FAF0F4）

## v2 初期スコープから除外する画面・機能

- イベント機能
- ドロップ残高

## モックデータ

各画面で使うモックデータの定義が必要:
- ユーザー情報（名前、アバター、レベル、スコア）
- アイドルグループ一覧
- クイズ問題（問題文、選択肢、正解、解説）
- ランキングデータ
- バッジデータ
- スコア推移データ
