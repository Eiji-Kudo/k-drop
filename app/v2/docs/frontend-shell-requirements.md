# v2 フロントエンド ガワアプリ 要件

## 目的

モックデータで全画面を繋ぎ、UI/UX の手触りを確認する。バックエンド API 接続は後のフェーズで行う。

## 技術スタック

| 技術 | 用途 |
|---|---|
| TanStack Router | ファイルベースルーティング |
| TanStack Query | データ取得（モックデータ） |
| React Hook Form + Zod | フォームバリデーション |
| Orval | **後回し**（API 接続時に導入） |

## デザイン方針

### v1 から引き継ぐ要素

- ピンク・紫系のカラーパレット（Primary: #FF9FCC, Accent: #A855F7, Background: #FAF0F4）
- カードベースのレイアウト
- グラデーション（ボタン、カード）
- 角丸（20-24px カード、8-10px ボタン）

### v2 で検討が必要な要素

- CSS フレームワーク / UI ライブラリの選定（Tailwind? shadcn/ui? 素 CSS?）
- PWA 向けのレスポンシブ対応（モバイルファースト、デスクトップ対応は？）
- アニメーションライブラリ（framer-motion?）
- チャートライブラリ（recharts?）

## モックデータ

各画面で使うモックデータの定義が必要:
- ユーザー情報（名前、アバター、レベル、スコア）
- アイドルグループ一覧
- クイズ問題（問題文、選択肢、正解、解説）
- ランキングデータ
- バッジデータ
- スコア推移データ
