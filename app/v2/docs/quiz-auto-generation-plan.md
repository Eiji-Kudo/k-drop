# v2 クイズ問題自動生成 計画

## 目的

K-POP グループごとのクイズ問題を LLM で自動生成し、コンテンツ量を効率的に確保する。

## 生成方法の候補

| 方法 | メリット | デメリット |
|---|---|---|
| **Claude API** | 高品質、日本語が得意 | API コスト |
| **OpenAI API** | 安定、エコシステム豊富 | API コスト |
| **ローカル LLM** | コスト $0 | 品質・速度のトレードオフ |

## 生成フローの候補

### A: CLI バッチ生成（推奨）

```
CLI ツール実行
  → グループ一覧取得
  → グループごとに LLM でクイズ生成
  → JSON / CSV 出力
  → 人間レビュー（ざっと確認）
  → D1 に INSERT
```

- 開発者が手元で実行
- 生成結果をファイルに保存してレビュー可能
- 問題があれば手動で修正してから投入

### B: GitHub Actions 定期実行

- cron で週次/月次に自動生成
- PR として差分を出す → レビュー → マージで投入
- 運用は楽だが、セットアップが重い

### C: 管理画面からオンデマンド

- 管理画面の「生成」ボタンで LLM を呼び出し
- プレビュー → 承認 → 保存
- 管理画面が必要

## プロンプト設計（草案）

```
あなたは K-POP に詳しいクイズ作家です。
以下のアイドルグループに関する4択クイズを {n} 問作成してください。

グループ名: {groupName}
難易度: {difficulty}

出力形式:
- prompt: 問題文
- choices: [{choiceText, isCorrect}] (4つ、正解は1つだけ)
- explanation: 解説

難易度の基準:
- easy: ファンなら誰でも知っている基本情報（メンバー名、デビュー年等）
- normal: ある程度のファンなら知っている情報（楽曲、活動歴等）
- hard: コアファンしか知らない情報（エピソード、裏話等）

注意:
- 事実に基づいた問題を作成すること
- 曖昧な問題や議論の余地がある問題は避けること
- 選択肢はもっともらしいものにすること
```

## 品質担保

- **自動検証**: 選択肢数 = 4、正解数 = 1、文字数制限チェック
- **重複チェック**: 既存問題との類似度チェック（簡易的にはプロンプト文のハッシュ比較）
- **人間レビュー**: 生成結果をファイル or PR で確認してから投入

## マスタデータ投入

クイズ生成の前提として、アイドルグループのマスタデータが必要:

- group_categories（ボーイズグループ、ガールズグループ等）
- idol_groups（BLACKPINK, BTS, TWICE, aespa, Stray Kids, IVE, LE SSERAFIM, NewJeans 等）

seed スクリプト or マイグレーションで投入。

## コスト見積もり

| 項目 | 見積もり |
|---|---|
| 1 問あたりのトークン数 | 入力 ~200 + 出力 ~300 = ~500 トークン |
| 100 問生成 | ~50,000 トークン |
| Claude Sonnet 想定 | ~$0.45 / 100 問 |
| 初期投入 500 問 | ~$2.25 |

月次で 100 問追加しても月 $0.5 未満。コスト的には問題なし。
