# v2 UI/UX ブラッシュアップ計画

- 作成日: 2026-04-05
- ステータス: Draft

## 目的

`app/v2` の UI を「daisyUI の素組み」から脱却させ、K-POP ファン向けの楽しさとモバイル PWA としての使いやすさを両立した体験に引き上げる。

今回の計画では、画面単位の見た目調整ではなく、先に共通デザイン基盤と UX ルールを作ってからホーム・クイズを中心に全体へ展開する。

モチベーション設計と報酬ループの詳細は [gamification-plan.md](/Users/eiji/program/k-drop/app/v2/docs/gamification-plan.md) を参照する。
ホームとクイズ結果に先に入れる最小構成の具体仕様は [motivation-ui-mvp-spec.md](/Users/eiji/program/k-drop/app/v2/docs/motivation-ui-mvp-spec.md) を参照する。

## ゴール

- 画面ごとにばらついている見出し・余白・カード・CTA の表現を統一する
- ピンク系のブランドカラーを活かしつつ、単調に見えない奥行きとメリハリを作る
- ホーム、クイズ、ランキング、プロフィールの 4 主要体験で「わくわく感」と「達成感」が出るようにする
- 主要導線でユーザーが迷わず次の行動を選べるようにする
- 操作結果、進行状況、選択状態が常に把握できるようにする
- 続けるほど気持ちよくなる報酬ループを UI 上で感じられるようにする
- モックデータ段階でも UI/UX の方向性を十分に検証できる状態にする

## 現状の課題

### 1. 共通デザインルールが不足している

- `index.css` はテーマ色の定義が中心で、背景、面の階層、シャドウ、タイポ、アクセント表現のルールが足りない
- ページ固有の Tailwind クラス記述が多く、画面ごとにトーンがずれやすい

### 2. 画面ごとの表現が分断している

- ホーム、ランキング、プロフィール、クイズでヘッダー構成と余白ルールが揃っていない
- カード、リスト、CTA の密度と強弱が画面ごとに異なる

### 3. K-POP アプリらしい感情演出が弱い

- クイズの正誤や進行に高揚感が少ない
- ランキングやプロフィールにコレクション感、誇らしさ、シェアしたくなる見せ場が少ない

### 4. 情報設計の階層が浅い

- 重要情報と補助情報の差が小さい
- 特にランキング、プロフィール、結果画面で視線誘導が弱い

### 5. 絵文字ベースの記号表現がプロトタイプ感を強めている

- ホームカードの絵文字は素人っぽく見えやすく、ブランドの信頼感を下げる
- タブでは `lucide-react` を使っているのに、ホームだけ絵文字でトーンが分断している
- 今後の UI では絵文字を情報アイコンとして使わず、アイコンシステムで統一する

### 6. UX 上の導線とフィードバックが弱い

- ホームで「まず何をすべきか」が弱く、主導線が十分に立っていない
- クイズ体験で現在位置、残り感、選択結果の理解が弱い
- ランキングとプロフィールで、自分に関係ある情報がひと目で分かりにくい
- フォーム画面で入力優先度、補助説明、完了後の安心感が不足している

### 7. 操作可能性の表現が弱い

- カード、タブ、選択肢、フォーム要素に「押せる」「選べる」「現在ここ」が十分に出ていない
- 非アクティブ状態と選択済み状態の差が小さく、判断コストが高い

### 8. ゲーミフィケーションの見せ方がほぼ存在しない

- レベル、成長、連続プレイ、ランキング変動、収集要素が UI 上で体験価値として見えていない
- クイズを解いた直後の「もう 1 問やりたい」を作る報酬の見せ方が弱い
- プロフィールやホームに、次に目指す目標や達成見込みが十分に出ていない

## デザイン方針

### 方向性

「Sweet Pop + Glossy Card」を基調にする。

- ベースはやわらかいピンク、アイボリー、チェリー系
- アクセントは紫を主役にしすぎず、差し色として限定的に使う
- 面はフラットよりも、薄いグラデーション、艶、ぼかし、柔らかい影で階層を出す
- 文字は見出しをやや大胆にし、本文は可読性優先で整理する

### 守るルール

- モバイルファースト。デスクトップでは中央寄せのまま、スマホ UI の密度を保つ
- 角丸、影、余白、見出しサイズはトークン化して再利用する
- ボタン、カード、タブ、フォーム、リストは共通プリミティブ経由で揃える
- アニメーションは少数精鋭に絞る。常時動かさず、CTA、遷移、正誤フィードバックに集中させる
- 絵文字は UI アイコンとして使わない。`lucide-react` などの線画アイコン、または将来的な専用アイコンで統一する

## UX 方針

### 体験原則

- 最初の 3 秒で「何のアプリか」「次に何をするか」が分かる
- 各画面で主 CTA は 1 つに寄せ、補助導線は視覚的に一段下げる
- 状態変化には必ずフィードバックを返す
- ユーザーの現在位置、選択内容、達成状況を見失わせない
- 入力フォームは「迷わせない」「戻りを少なくする」「完了を安心させる」を優先する

## ゲーミフィケーション方針

### 体験原則

- すべての主要画面で「今の達成」と「次の目標」が見える
- クイズ 1 回ごとに小さな報酬を返し、継続の理由を切らさない
- 数値だけでなく、称号、ランク、バッジ、進捗バーなど複数の報酬表現を使う
- ランキングは競争、プロフィールは蓄積、ホームは次の挑戦を促す役割に分ける
- 報酬演出は派手さよりも、続けたくなる手触りを優先する

### 導入したい報酬ループ

- [ ] クイズ開始前に「獲得できるもの」が見える
- [ ] 回答直後に正誤だけでなく、スコアや進捗の変化が返る
- [ ] 結果画面で成長実感や次の挑戦理由が提示される
- [ ] ホームで日次または短期の目標が見える
- [ ] プロフィールで収集と積み上げの価値が見える
- [ ] ランキングで競争の熱量が視覚的に伝わる

### 追加候補

- デイリーチャレンジ
- 連続プレイ日数
- 次レベルまでの残りポイント
- 新規バッジ獲得演出
- ランキング上昇幅の表示
- グループ別の習熟度表示

### UX チェック項目

- [ ] ホームで 1 タップ目の行動が明確か
- [ ] クイズ中に現在位置と残り感が分かるか
- [ ] 結果画面で成果と次アクションが直感的か
- [ ] ランキングで自分事化できる見せ方があるか
- [ ] プロフィールで蓄積の意味が理解できるか
- [ ] フォームで入力順とエラー修正方法が自然か
- [ ] タップ対象が十分な大きさと見た目を持っているか

## UX 観点の確認シナリオ

### ホーム

- [ ] 初見ユーザーが 3 秒以内に主機能を理解できる
- [ ] 最短 1 タップでクイズ開始導線に入れる
- [ ] 開いた瞬間に「今日やる意味」が伝わる

### クイズ

- [ ] グループ選択で迷わず 1 つ選べる
- [ ] 回答中に現在の進行状況を見失わない
- [ ] 回答直後に正誤と意味が分かる
- [ ] 終了後に次の選択肢が自然に提示される

### ランキング

- [ ] タブ切り替えの意味が初見でも分かる
- [ ] 自分の順位または注目ポイントをすぐ見つけられる
- [ ] 競争したい気持ちが自然に湧く

### プロフィール

- [ ] ユーザーの蓄積が一目で伝わる
- [ ] バッジや推しグループが単なる一覧ではなく成果として感じられる
- [ ] 次に目指す成長目標が分かる

### 作成フォーム

- [ ] 入力順に迷わない
- [ ] エラーが出ても修正箇所をすぐ特定できる
- [ ] 作成完了後の遷移と成功認知が自然である

## リスク

- 画面ごとに先に触ると、後から共通化し直す手戻りが大きい
- 色だけを派手にすると、ブランド感ではなく雑多さに見える
- クイズとホームを後回しにすると、全体の印象改善に対する効果が薄い
- 見た目だけ整えても、導線と状態設計が弱いままだと体験評価は上がらない
- ゲーミフィケーションを足しても、報酬の意味づけが弱いと安っぽい演出だけが残る
